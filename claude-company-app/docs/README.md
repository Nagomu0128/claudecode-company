# docs/ ガイド

このフォルダは、チーム全員が参照する情報の保管場所です。
「議論した内容」「判断理由」「仕様」「手順」など、あとで参照すべき情報はここに残します。

## 置き場所の目安
- `requirements.md`: 壁打ちで確定した要件定義（単一ソース）
- `decisions/`: 意思決定ログ（例: `YYYY-MM-DD-<topic>.md`）
- `api/`: API仕様や契約（エンドポイント一覧、リクエスト/レスポンス例）
- `design/`: 画面/導線、UI方針、コンポーネント方針
- `runbook/`: 実行手順、運用メモ、リリース手順

## 命名ルール(推奨)
- 日付+題名: `YYYY-MM-DD-<topic>.md`
- タイトルには「何が決まったか」が分かる語を入れる
